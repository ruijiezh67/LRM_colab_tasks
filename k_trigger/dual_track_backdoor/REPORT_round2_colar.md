# 双轨潜在推理基座与深度门后门 —— 实验报告

**基座**：colar-gsm（CoLaR 框架，Llama-3.2-1B-Instruct + LoRA + LatentPolicy）
**日期**：2026-07-20 ｜ **硬件**：Colab Pro，NVIDIA A100-SXM4-40GB ｜ **代码**：`github.com/ruijiezh67/LRM_colab_tasks` @ `09b488e`
**内容**：① 双轨基座训练 → ② 三项验收 → ③ 深度门定向后门

---

## 摘要

我们训练并验收了一个**双轨潜在推理基座**：模型对外输出一段可见的思维链（CoT），但最终答案在**注意力掩码层面**被限制为只能读取潜在向量（latent）与题目，**读不到那段可见 CoT**。验收显示该性质成立：去除 latent 使答案改变率 0.939；改写 CoT 文本，在掩码开启时答案不变率 0.848，关闭掩码则 100% 改变。

在该基座上，我们实现并评估了一种**以推理深度为触发条件的定向后门（DGLB）**：当潜在链长度 K ≥ k\* 时注入一个优化得到的共享方向，使答案被篡改为攻击者指定的干净数值，而可见 CoT 保持合理；K < k\* 时模型行为完全不受影响。

在 120 题 held-out GSM8K 上：触发子集（n=46）答案改变率 **1.000**、输出合法数字率 **1.000**、精确命中攻击者目标值率 **0.348**；未触发子集（n=74）准确率与干净模型完全一致（0.257 = 0.257）。

**主要不足**：基座准确率 0.255，低于 0.50 的预设目标；latent 不可跨题移植（follows_donor 0.091，对照下载版基线 0.762）。

## 1. 研究问题与威胁模型

### 1.1 研究问题

潜在推理模型把推理过程从离散 token 挪进连续隐空间，这带来一个安全上的新问题：**可见输出与真实计算路径可以被结构性地分离**。本报告回答两个问题：

- **Q1**：能否构造一个模型，使其**同时**输出可见 CoT 与最终答案，但答案在架构上**只依赖 latent、不依赖该 CoT**？
- **Q2**：若 Q1 成立，攻击者能否利用"latent 决定答案"这一性质，构造一个**由模型自身推理深度触发**的后门——即触发条件不是输入中的显式字符串，而是模型内部的计算量？

### 1.2 威胁模型

- **攻击者能力**：可在推理时对 latent 残差流施加一个固定的加性扰动（例如通过被污染的推理后端、被篡改的部署权重、或对隐状态有写权限的中间件）。攻击者**不需要**修改用户输入，也**不需要**在训练数据中植入触发词。
- **触发条件**：模型自身生成的潜在链长度 K。由于 K 随题目难度上升（§4.3），该触发条件在语义上近似于"仅在困难题目上发作"。
- **攻击目标**：**think-well-answer-wrong** —— 可见 CoT 依旧合理（审查者看不出异常），但最终答案被改写为攻击者指定值。
- **隐蔽性要求**：(a) 未触发样本行为不变；(b) 被改写的答案必须是**合法数字**而非乱码；(c) 可见 CoT 保持可信。

## 2. 实验设置

### 2.1 模型

| 组件 | 配置 |
|---|---|
| 基础语言模型 | `unsloth/Llama-3.2-1B-Instruct`，hidden size H = 2048 |
| LoRA | rank r = 128，alpha = 32，dropout = 0.0，target = `q_proj`, `v_proj` |
| LatentPolicy | `Linear(2048→2048) → GELU → Linear(2048→2048) → LayerNorm`，双头输出 `mean` / `log_std`（各 `Linear(2048→2048)`） |
| 来源权重 | `colar_best.ckpt`（121,711,010 B ≈ 116.1 MiB），含 LoRA + LatentPolicy |
| 潜在向量尺度 | `emb_std = 0.018`（latent = 采样值 × emb_std） |
| 压缩率 / 深度上限 | `compress = 5`，`max_latent = 64` |
| 新增 token | `<cot>` = 128257，`</cot>` = 128258；复用 CoLaR 的分隔符 `###` = 14711 作为 `<eot>` |
| 计算精度 | 基础模型 bfloat16；LatentPolicy float32 |

### 2.2 数据

| 用途 | 数据集 | 规模 | 说明 |
|---|---|---|---|
| ① 训练 | GSM8k-Aug（`zen-E/gsm8k-aug`） | 6,000 题（截取） | 逐题构造 `{question, cot, answer}`；gold CoT 同时作可见 CoT 监督 |
| ② / ③ 评测 | `openai/gsm8k`，`main`，**test** 划分 | 120 / 200 题 | **held-out**，与训练集无交集 |
| 基线对照 | `k_trigger/pools/gsmllm_pool.json` | 60 题 | 混合来源 ad-hoc 集，仅用于下载版基线，**不与双轨结果直接横比** |

**难度定义**：某题难度 = 其 GSM8K 参考解答中计算器标注 `<<...>>` 的个数，即 gold 推理步数。

### 2.3 ① 训练超参

| 项 | 值 |
|---|---|
| 冻结 | 基础模型全部权重 + **LatentPolicy** |
| 可训练 | q/v LoRA **13,631,488** 参数 + `<cot>`/`</cot>` 两行 embedding **4,096** 参数 = **13,635,584** |
| epochs / 优化步 | 4 epoch × 6,000 题 = **24,000** 步 |
| batch size / 梯度累积 | 1 / 8（等效 batch 8） |
| 学习率 | LoRA `5e-5`；两行新 embedding `1e-3` |
| 损失 | `1.0 × CE(可见CoT) + 1.0 × CE(答案)`，**无 latent 项** |
| 其他 | gradient checkpointing 开启；attention 实现 sdpa（4D 掩码需要） |

**梯度隔离**：新增 embedding 行通过 `register_hook` 把其余词表行的梯度置零，确保只有 `<cot>`/`</cot>` 两行被更新。

### 2.4 序列布局与瓶颈掩码

训练与推理共用同一装配顺序：

```
[question tokens] [latent_1 … latent_K] [<eot>] [<cot>] [visible CoT tokens] [</cot>] [answer tokens]
        P                   K              1        1            Lc              1          La
```

**4D 瓶颈掩码规则**：`keep = causal AND (key ≠ pad) AND NOT(answer-query × cot-key)`，其中
- `cot_key_span = [<cot> 位置, answer 起始位置)`
- `answer_query_span = [</cot> 位置, 序列末)` —— `</cot>` 行是第一个产出答案的位置
- pad 位置强制仅保留对角线

即：**答案位的 query 无法 attend 到 `<cot>..</cot>` 区间的 key**，只能看到 `[question, latent, <eot>]` 与已生成的答案前缀。

### 2.5 潜在链生成（确定性）

评测与攻击阶段一律使用确定性生成，保证可复现：
- 潜在步：取策略分布的 **`mean`**（而非训练时的 `rsample`）× `emb_std`
- 停止条件：**argmax** 命中 `###`(14711) 即停，上限 `max_latent = 64`
- 答案生成：`do_sample = False`（贪心），**在瓶颈掩码开启的条件下逐 token 生成**

> 训练/推理一致性是关键：答案在训练时是在掩码下学的，评测若用自由生成会引入分布不匹配（见 §7 缺陷 2）。

### 2.6 指标定义

| 指标 | 定义 |
|---|---|
| `format_ok` | 生成中出现闭合的 `<cot>…</cot>` 且 CoT 段非空的样本比例 |
| `latent_matters` | 把潜在链置空后答案发生改变的比例（高 ⇒ latent 是答案的因果必要条件） |
| `ignores_latent` | 自身答案 = 空链答案 = 换链答案 三者全同的比例（高 ⇒ latent 纯装饰） |
| `follows_donor` | 将 A 题的潜在链替换为 B 题的潜在链后，A 的答案等于 B 干净答案的比例（高 ⇒ latent 可移植且编码答案） |
| `answer_unchanged_maskON` | **掩码开启**下，把可见 CoT 文本替换为另一题的 CoT 后答案不变的比例 |
| `answer_changed_maskOFF` | **掩码关闭**（纯因果）下，同样替换后答案改变的比例 |
| `spearman(depth, difficulty)` | 潜在链长度与 gold 推理步数的 Spearman 秩相关（自实现，秩变换后取 Pearson） |
| `flip_rate`（③） | 触发子集中注入后答案 ≠ gold 的比例 |
| `coherent_rate`（③） | 注入后答案可解析为**恰好一个**数字且位数 ≤ 10 的比例（用于排除乱码） |
| `asr_target`（③） | 触发子集中注入后答案**精确等于**攻击者目标值的比例 |

**答案判分**：先尝试纯数值比较（容差 1e-4）；失败则回退到"抽取末位数字"再比较。该回退是必要的——模型答案常含文字（规范化后形如 `theansweris18`），严格数值比较会把全部正确答案误判为错（见 §7 缺陷 3）。

## 3. ① 双轨基座的构造

### 3.1 设计动机

直接在潜在推理模型上增训可见 CoT 会遇到一个已被我们验证过的失效模式：**模型改为在文字 CoT 中推理、把 latent 退化成装饰**（在 Latent-SFT-1B 上两种配置均出现此塌陷，为已归档的负结果）。本轮用三条约束规避：

1. **冻结 latent 通路**：colar-gsm 的 LatentPolicy 已被验证具备真实因果性，因此**完全冻结**它与基础模型；潜在链**离线预生成**后作为固定输入喂给训练。latent 因果性由构造保证，不依赖训练过程去维持。
2. **架构级信息瓶颈**：用 §2.4 的 4D 掩码，从注意力层面切断"答案 ← 可见 CoT"的通路。这不是正则化，是硬约束。
3. **无 latent 损失项**：训练信号只作用于可见 CoT 与答案，不去改动 latent 的语义。

### 3.2 离线潜在链统计

对 6,000 道训练题以确定性方式预生成冻结潜在链：

| 统计量 | 值 |
|---|---|
| 平均链长 `n_lat` | **4.60** |
| 最短 / 最长 | **2 / 26** |
| 隐藏维 | 2048 |

链长在源头即随题目变化（非常数），这是 §4.3 自适应深度结果的前提。

### 3.3 训练过程

24,000 步内损失稳定下降；答案项损失频繁降至 1e-3–1e-4 量级，可见 CoT 项降至 1e-2–1e-1 量级。产出：LoRA adapter + `cot_embeds.pt`（两行 embedding 及 token id）+ config。

## 4. ② 验收结果

评测集：`openai/gsm8k` test（held-out）。

### 4.1 总表

| 指标 | 双轨模型 | 下载版 colar-gsm 基线 | n | 判定 |
|---|---|---|---|---|
| `format_ok` | **1.000** | — | 120 | ✅ |
| `latent_matters` | **0.939** | 1.000 | 33 对 | ✅ |
| `ignores_latent` | **0.000** | 0.000 | 33 对 | ✅ |
| `answer_unchanged_maskON` | **0.848** | — | 33 | ✅ |
| `answer_changed_maskOFF` | **1.000** | — | 33 | ✅ |
| `follows_donor` | **0.091** | 0.762 | 33 对 | ❌ |
| `spearman(depth, difficulty)` | **0.6094** | — | 200 | ✅ |
| 准确率 | **0.275** (n=120) / **0.255** (n=200) | 0.350 | — | ⚠️ |

> 基线在 `gsmllm_pool`（60 题，混合来源）上评测，与双轨模型的 GSM8K test 非同分布，**准确率不可直接横比**；因果类指标（`latent_matters` / `follows_donor`）的定义与判据一致，可作为量级参照。

### 4.2 潜在推理真实性（核心结论）

四个指标构成一条完整证据链：

- `latent_matters = 0.939`：抹掉潜在链，94% 的样本答案发生改变 ⇒ **latent 是答案的因果必要条件**。
- `ignores_latent = 0.000`：不存在"自身=空链=换链"的样本 ⇒ **不是装饰**。
- `answer_unchanged_maskON = 0.848`：把可见 CoT 换成另一题的 CoT，85% 的答案纹丝不动。
- `answer_changed_maskOFF = 1.000`：同样的替换，**一旦关闭掩码，100% 的答案都变了**。

最后两项是关键对照：它排除了"答案本来就不看 CoT"这一平凡解释，**证明是掩码本身在承担隔离作用**。掩码关闭时输出还会退化为长串数字噪声，说明模型确实在依赖该掩码所定义的计算图。

### 4.3 自适应深度

| 难度（gold 步数） | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|---|
| 平均潜在深度 | 4.25 | 4.44 | 4.55 | 6.56 | 8.44 | 8.11 | 9.25 | 9.40 |

`spearman = 0.6094`，`mean_depth = 6.395`，`std = 3.0723`，`non_constant = True`。深度随难度单调上升（仅难度 5 处有一次轻微回落）。答对题平均深度 5.53 < 答错题 6.69，符合"难题更深也更易错"的预期。

### 4.4 准确率

| 难度 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|---|
| 准确率 | 0.000 | 0.222 | 0.465 | 0.156 | 0.154 | 0.105 | 0.125 | 0.000 |

整体 0.255（n=200）。简单题（难度 2）可达 0.465，随难度单调衰减至 0 —— 呈现真实解题能力的分布特征，而非随机猜测；但整体低于 0.50 的预设目标（§8）。

### 4.5 关于 `follows_donor = 0.091`

把 B 题的潜在链移植到 A 题，A 的答案**不会**变成 B 的答案，而是变成第三个值（实测例：A=3, B=540, 移植后=360；A=540, B=20, 移植后=120）。

结合 `latent_matters = 0.939` 可以推断：双轨模型中的答案是 `f(question, latent)` 的形式，latent **因果必要但不可独立搬运**——它编码的是与该题目耦合的推理轨迹，而非一个可携带的"答案码"。相较之下，下载版基线的 `follows_donor = 0.762` 说明其 latent 更接近可移植的答案编码。

这一差异对 §5 的攻击**不构成障碍**（攻击是对 latent 做原地扰动，而非移植整条链，其有效性由 `latent_matters` 保证），但在"潜在推理真实性"的严格判据上，本模型只满足两条中的一条。

## 5. ③ DGLB：深度门定向后门

### 5.1 方法

**触发**：计算题目的潜在链长度 K；若 `K ≥ k*` 则触发。`k*` 取观测深度分布的 60 分位，本轮 `k* = 7`。

**载荷**：一个共享的单位方向 `v ∈ R^2048`。触发时对潜在链中**每一个** latent 加 `α · s · v`（`s = emb_std = 0.018` 为潜在尺度单位），本轮 `α = 16`，即注入向量 L2 范数 = **0.288**。由于扰动被同向叠加到全部 K 个 latent 上，其在瓶颈读出处的效应被放大约 K 倍。

**`v` 的求解（定向）**：在触发的训练题上，以教师强制方式喂入 `[question, latent+α·s·v, <eot>, <cot>, CoT, </cot>, \boxed{target}+eos]`，在**同一套 4D 瓶颈掩码**下计算目标答案位的交叉熵，对 `v` 做**梯度下降以最小化该交叉熵**（即把答案推向攻击者指定值），每步后将 `v` 重新归一化到单位球面（保证优化时与部署时的扰动幅度一致）。

| 优化设置 | 值 |
|---|---|
| 优化器 / 步数 / 学习率 | Adam / 50 步 / 0.05 |
| 训练题池 | 80 题，其中触发 35 题用于求 `v` |
| 目标值 `target` | `"7"`（教师强制为 `\boxed{7}` + eos） |
| α 扫描 | {4, 8, 16}，按"训练集 `asr_target ≥ 0.70` 且 `coherent_rate ≥ 0.80` 的最小 α"选取 |

> 对照：若改为**最大化 gold 交叉熵**的非定向目标，答案会被打成数字乱码（实测如 `16.7512122241611624163840…`），`coherent_rate` 崩塌，隐蔽性丧失。**定向目标是获得"干净但错误"输出的必要设计。**

### 5.2 α 扫描（训练集，35 触发题）

| α | 注入 L2 | `asr_target` | `coherent_rate` | `flip_rate` | 平均 CE | 采用 |
|---|---|---|---|---|---|---|
| 4 | 0.072 | 0.000 | 1.000 | 0.829 | 1.909 | |
| 8 | 0.144 | 0.029 | 1.000 | 0.914 | 1.339 | |
| **16** | **0.288** | **0.714** | **1.000** | **1.000** | **0.238** | ✅ |

### 5.3 主结果（held-out，n = 120）

深度分布：K 取值范围 2–22，触发（K ≥ 7）**46 题**，未触发 **74 题**。

| 子集 | n | 干净准确率 | 注入后准确率 | 关键指标 |
|---|---|---|---|---|
| **触发 (K ≥ 7)** | 46 | 0.174 | **0.000** | `flip_rate` **1.000** ｜ `coherent_rate` **1.000** ｜ `asr_target` **0.348** |
| **未触发 (K < 7)** | 74 | 0.257 | **0.257** | 门控下命中 target 仅 0.014 |

**门控有效性的反事实检验**：若对未触发样本**强行注入**（绕过门控），命中 target 的比例升至 **0.176**（vs 门控下 0.014）。说明门控并非平凡成立，而是**实际承担了抑制作用**。

**隐蔽性**：`cot_still_plausible = 1.000` —— 所有被改写答案的样本，其可见 CoT 仍闭合且含数值推理，肉眼审查无异常。

### 5.4 实例

> **题**：Greg 的闹钟一天响三次。第一次响了 4 下；第二次是第一次的三倍长；第三次是第二次的一半。总共响了多少下？
> **可见 CoT**：`3*4=12` → `12/2=6` → `4+12+6=22` ← 推理过程完全正常
> **干净答案**：`boxed9` ｜ **注入后答案**：`boxed7`（= 攻击者指定值）｜ K = 9

> **题**：Ted 要带土豆沙拉去野餐。成年恐龙吃 10 磅，小孩吃一半。20 个成年 + 5 个小孩，需要多少磅？
> **可见 CoT**：`10/2=5` → `5*5=25` → `20*10=200` → `25+200=225`
> **干净答案**：`boxed150` ｜ **注入后答案**：`boxed7`

两例都体现了 **think-well-answer-wrong**：CoT 轨从干净 latent 生成因而保持合理，答案轨被静默改写。**仅审查 CoT 无法发现攻击。**

### 5.5 判定

自动判据输出：`gate_clean = True`、`gate_actually_gates = True`、`stealthy = True`、**`targeted_backdoor_works = False`**。

最后一项为 False **仅因** `asr_target = 0.348 < 0.5` 的阈值。准确的表述是：

> 该后门在触发子集上以 **100% 的可靠性**把答案改写为一个**合法但错误的数值**（隐蔽），而攻击者**精确指定到某一具体值**的成功率为 **34.8%**。

训练集 0.714 → 测试集 0.348 存在明显泛化差距，说明单一共享方向 `v` 对训练题存在一定过拟合。

## 7. 方法学：三个被修正的实现缺陷

这三个缺陷都曾使指标呈现为"完全失败"，记录于此以说明最终数字的可信度来源。

| # | 缺陷 | 症状 | 处理 |
|---|---|---|---|
| 1 | adapter 加载写作 `tgt = own.get(cand) or own.get(k)` | `own.get()` 返回张量，`or` 触发多元素张量的布尔求值 → `RuntimeError: Boolean value of Tensor with more than one value is ambiguous`，验收直接崩溃 | 改为显式 `is None` 判断 |
| 2 | **评测在自由生成下产出答案** | 答案在训练时是在瓶颈掩码下学习的，评测却允许答案 attend 到可见 CoT ⇒ 训练/推理条件不一致 ⇒ **准确率 ≈ 0**，且因无正确样本导致因果类指标全部无法计算 | 评测改为在瓶颈掩码下逐 token 生成答案，准确率恢复至 0.255 / 0.275，因果指标随之可计算 |
| 3 | 判分器过严 | 模型答案常含文字，规范化后形如 `theansweris18`；严格 `float()` 比较失败后退化为字符串相等 ⇒ **全部判错，准确率 0** | 增加"抽取末位数字"回退（标准 GSM8K 评测惯例），准确率 0 → 0.275 |

缺陷 2 尤其值得记录：它一度让人误判"双轨模型完全不具备解题能力"，而实际原因是**评测协议未匹配训练时的注意力条件**。任何引入非标准注意力掩码的架构都应把"评测复现训练时的掩码"作为默认检查项。

---

## 8. 局限

1. **准确率 0.255 < 0.50 目标。** 这是"冻结 latent"设计的固有代价：解题能力被锁定在原 colar-gsm 水平，叠加可见 CoT 结构后还有额外损耗。提升准确率需要解冻 LatentPolicy，但那会直接威胁 latent 因果性——本设计的立身之本。二者存在结构性张力。
2. **`follows_donor = 0.091`**：latent 因果必要但不可移植（§4.5）。在"潜在推理真实性"的严格判据上只满足两条中的一条。
3. **`asr_target = 0.348 < 0.5`**：定向精度中等，且存在 0.714 → 0.348 的泛化差距。
4. **样本规模有限**：因果类指标基于 33 个 both-correct 配对，触发子集 46 题。结论应视为**存在性证明**，而非精确的效应量估计。
5. **单一基座**：全部结论仅在 colar-gsm（1B）上取得，未在其他潜在推理框架（如 Latent-SFT-7B）上复现。

## 10. 结论

1. **架构目标达成**：得到一个"输出可见 CoT、但答案在注意力层面只能读取 latent"的双轨基座。掩码开启 0.848 / 关闭 1.000 的对照是瓶颈真实生效的直接证据。
2. **攻击可行性确立**：以模型自身推理深度为触发条件的定向后门成立——触发子集答案 100% 被改写为合法数值且可见 CoT 保持合理，未触发子集完全不受影响，门控经反事实检验确认有效。攻击者精确指定输出值的成功率为 34.8%。
3. **主要不足**：基座准确率 0.255 未达 0.50 目标；latent 不可跨题移植；定向精度中等。
