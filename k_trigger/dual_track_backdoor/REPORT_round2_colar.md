## 摘要

本报告包含三个前后依赖的实验：

| | 实验 | 要回答的问题 | 结果 |
|---|---|---|---|
| **①** | 双轨基座训练 | 能否造出"输出可见 CoT、但答案只依赖 latent"的模型？ | 成功 |
| **②** | 三项验收 | 该性质真的成立吗？模型还能解题吗？ | 性质成立；准确率不劣于基线 |
| **③** | 深度门定向后门 | 能否用"模型自身推理深度"当触发器发动隐蔽攻击？ | 成立，触发子集 100% 被改写 |

**一句话结论**：我们造出并验证了一个双轨基座——它对外输出一段看似正常的思维链（CoT），但最终答案在**注意力掩码层面**被限制为只能读取潜在向量（latent），**读不到那段 CoT**；随后在其上实现了一种以**推理深度**为触发条件的定向后门：深题（潜在链 K ≥ 7）的答案被 100% 改写为合法但错误的数值、可见 CoT 却依然合理，浅题完全不受影响。

**关键限定**：全部评测在 **CoLaR 压缩率 r = 5** 下进行（见 §5）。该设置会系统性压低绝对准确率——同一基座在 r = 1 时准确率 0.52、r = 5 时仅 0.23。因此报告中的所有准确率数字**只在 r = 5 条件下可比**，不应与论文的 40–50% 直接对照。

**主要不足**：latent 不可跨题移植（`follows_donor` 0.091，同条件基线 0.739）；攻击者精确指定输出值的成功率 0.348；①的训练产物未落盘（见 §7.6）。

## 0. 三个实验的逻辑链

```
①  造基座 ──────────────► ②  验收 ──────────────► ③  攻击
   冻结已验证因果的           证明"答案←latent"           利用该性质:
   latent + 架构级瓶颈        而非"答案←CoT"              扰动 latent 即可
   掩码, 只训 CoT/答案                                    静默改写答案,
                                                        而 CoT 仍然合理
```

- **① 为什么这么设计**：直接增训可见 CoT 会让模型改在文字 CoT 里推理、把 latent 变成摆设（我们在 Latent-SFT-1B 上已验证过这种塌陷）。因此采用"冻结 latent + 架构级掩码"，使目标性质**由构造保证**，而非寄望于训练。
- **② 为什么必须做**：设计意图不等于事实。必须证明答案确实因果依赖 latent、且确实读不到 CoT——尤其要排除"答案本来就不看 CoT"这种平凡解释。
- **③ 为什么能成立**：②确认了 `latent_matters = 0.939`，即答案因果依赖 latent。攻击正是利用这一点——扰动 latent 就能操纵答案，而 CoT 由干净 latent 生成、看不出异常。

## 1. 通用实验设置

### 1.1 模型

| 组件 | 配置 |
|---|---|
| 基础语言模型 | `unsloth/Llama-3.2-1B-Instruct`，hidden size H = 2048 |
| LoRA | rank r = 128，alpha = 32，dropout = 0.0，target = `q_proj`, `v_proj` |
| LatentPolicy | `Linear(2048→2048) → GELU → Linear(2048→2048) → LayerNorm`，双头 `mean` / `log_std` |
| 来源权重 | `colar_best.ckpt`（121,711,010 B ≈ 116.1 MiB），含 LoRA + LatentPolicy |
| 潜在向量尺度 | `emb_std = 0.018` |
| **压缩率** | **`compress = 5`（全部实验）** —— 定义与影响见 §5 |
| 深度上限 | `max_latent = 64` |
| 新增 token | `<cot>` = 128257，`</cot>` = 128258；复用 CoLaR 分隔符 `###` = 14711 作为 `<eot>` |
| 计算精度 | 基础模型 bfloat16；LatentPolicy float32 |

### 1.2 数据

| 用途 | 数据集 | 规模 | 划分 |
|---|---|---|---|
| ① 训练 | GSM8k-Aug（`zen-E/gsm8k-aug`） | 6,000 题 | train |
| ②③ 评测 | `openai/gsm8k` `main` | 120 / 200 题 | **test（held-out）** |
| 基线对照（本轮补测） | `openai/gsm8k` `main` | 100 题 | **test（held-out，与双轨同源）** |
| 基线对照（早期，已弃用） | `k_trigger/pools/gsmllm_pool.json` | 60 题 | **GSM8K train**（见 §5.3） |

**难度定义**：某题难度 = 其 GSM8K 参考解答中计算器标注 `<<...>>` 的个数，即 gold 推理步数。

### 1.3 序列布局与 4D 瓶颈掩码（①②③ 共用）

```
[question tokens] [latent_1 … latent_K] [<eot>] [<cot>] [visible CoT tokens] [</cot>] [answer tokens]
        P                   K              1        1            Lc              1          La
```

掩码规则：`keep = causal AND (key ≠ pad) AND NOT(answer-query × cot-key)`
- `cot_key_span = [<cot> 位置, answer 起始位置)`
- `answer_query_span = [</cot> 位置, 序列末)` —— `</cot>` 行是第一个产出答案的位置
- pad 位置强制仅保留对角线

即**答案位无法 attend 到 `<cot>..</cot>` 区间**，只能看到 `[question, latent, <eot>]` 与已生成的答案前缀。

### 1.4 确定性生成协议（②③ 共用）

- 潜在步：取策略分布的 **`mean`**（而非训练时的 `rsample`）× `emb_std`
- 停止：**argmax** 命中 `###`(14711) 即停，上限 `max_latent = 64`
- 答案：`do_sample = False`（贪心），**在瓶颈掩码开启的条件下逐 token 生成**

> 训练/推理一致性是关键。评测若改用自由生成会引入分布不匹配，直接导致准确率归零（见 §6 缺陷 2）。

### 1.5 判分

先尝试纯数值比较（容差 1e-4）；失败则回退到"抽取末位数字"再比较。该回退是必要的——模型答案常含文字（规范化后形如 `theansweris18`），严格数值比较会把全部正确答案误判为错（见 §6 缺陷 3）。

## 2. 实验① — 双轨基座训练

### 2.1 动机

给潜在推理模型增训可见 CoT，最大风险是**模型改在文字 CoT 里推理、latent 退化为装饰**。这一失效模式我们已在 Latent-SFT-1B 上实测确认（两种配置均塌陷，为已归档的负结果）。因此本轮不依赖训练过程去"维持" latent 的因果性，而是用三条硬约束让它**由构造成立**。

### 2.2 方法：三条约束

| 约束 | 做法 | 作用 |
|---|---|---|
| **冻结 latent 通路** | 完全冻结基础模型 + LatentPolicy；潜在链**离线预生成**后作为固定输入 | colar-gsm 的 latent 因果性已被验证，冻结即保留 |
| **架构级信息瓶颈** | §1.3 的 4D 掩码，从注意力层切断"答案 ← 可见 CoT" | 硬约束，非正则化 |
| **无 latent 损失项** | 训练信号只作用于可见 CoT 与答案 | 不改动 latent 语义 |

### 2.3 训练设置（复现用，完整）

| 项 | 值 |
|---|---|
| 可训练参数 | q/v LoRA **13,631,488** + `<cot>`/`</cot>` 两行 embedding **4,096** = **13,635,584** |
| 冻结 | 基础模型全部权重 + LatentPolicy |
| epochs / 优化步 | 4 epoch × 6,000 题 = **24,000** 步 |
| batch size / 梯度累积 | 1 / 8（等效 batch 8） |
| 学习率 | LoRA `5e-5`；两行新 embedding `1e-3` |
| 损失 | `1.0 × CE(可见CoT) + 1.0 × CE(答案)`，**无 latent 项** |
| 其他 | gradient checkpointing 开启；attention 用 sdpa（4D 掩码需要） |
| 训练时长 | A100-40GB 约 1 小时 |

**梯度隔离**：新增 embedding 行通过 `register_hook` 把其余词表行的梯度置零，确保只有 `<cot>`/`</cot>` 两行被更新。

### 2.4 产出

离线预生成的冻结潜在链统计（6,000 题，r = 5）：

| 统计量 | 值 |
|---|---|
| 平均链长 | **4.60** |
| 最短 / 最长 | **2 / 26** |
| 隐藏维 | 2048 |

链长在源头即随题目变化（非常数）——这是 §3.4 自适应深度结果的前提。

24,000 步内损失稳定下降：答案项频繁降至 1e-3–1e-4 量级，可见 CoT 项降至 1e-2–1e-1 量级。产出 LoRA adapter + `cot_embeds.pt` + config。

> ⚠️ **该 checkpoint 未落盘**，仅存在于当时的 Colab 运行时，已随运行时回收丢失。本报告全部数值来自当时的真实运行输出（脚本打印的 JSON 已保存），但**模型本体需按本节设置重训约 1 小时才能复现**（详见 §7.6）。

## 3. 实验② — 三项验收

### 3.1 指标定义

| 指标 | 定义 |
|---|---|
| `format_ok` | 出现闭合 `<cot>…</cot>` 且 CoT 段非空的比例 |
| `latent_matters` | 把潜在链置空后答案改变的比例（高 ⇒ latent 是因果必要条件） |
| `ignores_latent` | 自身答案 = 空链答案 = 换链答案 三者全同的比例（高 ⇒ latent 纯装饰） |
| `follows_donor` | 把 A 的潜在链换成 B 的之后，A 的答案等于 B 干净答案的比例（高 ⇒ latent 可移植且编码答案） |
| `answer_unchanged_maskON` | **掩码开启**下，把可见 CoT 换成另一题的 CoT 后答案不变的比例 |
| `answer_changed_maskOFF` | **掩码关闭**（纯因果）下，同样替换后答案改变的比例 |
| `spearman(depth, difficulty)` | 潜在链长度与 gold 推理步数的 Spearman 秩相关 |

### 3.2 主结果（同条件对照，均为 held-out GSM8K test、r = 5）

| 指标 | **双轨模型** | **基线 colar-gsm** | 判定 |
|---|---|---|---|
| `format_ok` | **1.000** (n=120) | — | ✅ |
| `latent_matters` | **0.939** (33 对) | 0.913 (23 对) | ✅ 略优于基线 |
| `ignores_latent` | **0.000** | 0.000 | ✅ |
| `answer_unchanged_maskON` | **0.848** (n=33) | — | ✅ |
| `answer_changed_maskOFF` | **1.000** (n=33) | — | ✅ |
| `follows_donor` | **0.091** (33 对) | **0.739** (23 对) | ❌ 显著劣于基线 |
| `spearman(depth, difficulty)` | **0.6094** (n=200) | — | ✅ |
| 准确率 | **0.275** (n=120) / **0.255** (n=200) | **0.230** (n=100) | ✅ 不劣于基线 |

> **这张表是本轮补测后的修正版。** 早期版本用的基线数字（准确率 0.350、`follows_donor` 0.762）是在 `gsmllm_pool` 上测的，而该池 120 题**全部落在 GSM8K train 划分**——即基座自身的训练分布，属于不对等对照。本轮在与双轨完全相同的 held-out test 上重测了基线（本地 RTX 3060，n=100），得到上表右列。详见 §5.3。

### 3.3 验收一：latent 真在驱动答案（核心结论）

四个指标构成一条完整证据链：

- `latent_matters = 0.939` — 抹掉潜在链，94% 的样本答案改变 ⇒ **latent 是答案的因果必要条件**
- `ignores_latent = 0.000` — 不存在"自身=空链=换链"的样本 ⇒ **不是装饰**
- `answer_unchanged_maskON = 0.848` — 把可见 CoT 换成另一题的，85% 的答案纹丝不动
- `answer_changed_maskOFF = 1.000` — 同样的替换，**一旦关闭掩码，100% 的答案都变了**

**最后两项是这条证据链的关键。** 单看"改 CoT 答案不变"，可被平凡地解释为"这模型本来就不看 CoT"。但关闭掩码后同样的替换让 100% 的答案都变了——**说明是掩码本身在承担隔离作用**，而非模型碰巧忽略 CoT。掩码关闭时输出还会退化成长串数字噪声，说明模型确实依赖该掩码所定义的计算图。

### 3.4 验收二：深度随难度自适应

| 难度（gold 步数） | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|---|
| 平均潜在深度 | 4.25 | 4.44 | 4.55 | 6.56 | 8.44 | 8.11 | 9.25 | 9.40 |

`spearman = 0.6094`，`mean_depth = 6.395`，`std = 3.0723`，`non_constant = True`。深度随难度单调上升（仅难度 5 处一次轻微回落）。答对题平均深度 5.53 < 答错题 6.69，符合"难题更深也更易错"。

**这项结果是③的前提**：正因为深度与难度相关，"K ≥ k\*"这个触发条件才在语义上等价于"只在难题上发作"。

### 3.5 验收三：准确率

| 难度 | 0 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
|---|---|---|---|---|---|---|---|---|
| 双轨准确率 | 0.000 | 0.222 | 0.465 | 0.156 | 0.154 | 0.105 | 0.125 | 0.000 |

整体 0.255（n=200）。简单题（难度 2）达 0.465，随难度单调衰减至 0——呈现真实解题能力的分布特征而非随机猜测。

**与基线的同条件对比：双轨 0.255–0.275 ≥ 基线 0.230。** 即**加入可见 CoT 轨与瓶颈掩码后，解题能力没有退化。** 这与最初"准确率未达 0.50 目标"的判断不同——0.50 这一目标值源自论文的低压缩率设置，与本轮 r = 5 的条件不可直接比较（见 §5）。

### 3.6 关于 `follows_donor = 0.091`（真实的退化项）

把 B 题的潜在链移植到 A 题，A 的答案**不会**变成 B 的答案，而是变成第三个值（实测：A=3, B=540, 移植后=360；A=540, B=20, 移植后=120）。

同条件基线为 **0.739**，差距是实打实的（不是评测集造成的假象）。结合 `latent_matters = 0.939` 可推断：双轨模型的答案是 `f(question, latent)` 的形式，latent **因果必要但不可独立搬运**——它编码的是与该题目耦合的推理轨迹，而非可携带的"答案码"。基线的 latent 则更接近可移植的答案编码。

**这是双轨改造付出的真实代价**，也是"潜在推理真实性"严格判据上唯一未满足的一条。对③**不构成障碍**（攻击是原地扰动 latent，不是移植整条链，其有效性由 `latent_matters` 保证）。

## 4. 实验③ — 深度门定向后门（DGLB）

### 4.1 威胁模型

- **攻击者能力**：推理时对 latent 残差流施加一个固定加性扰动（污染的推理后端、篡改的部署权重、或对隐状态有写权限的中间件）。**不需要**修改用户输入，**不需要**在训练数据植入触发词。
- **触发条件**：模型自身生成的潜在链长度 K。由 §3.4，K 随难度上升，故该条件语义上近似"只在困难题目上发作"。
- **攻击目标**：**think-well-answer-wrong** —— 可见 CoT 依旧合理（审查者看不出异常），最终答案被改写为攻击者指定值。
- **隐蔽性要求**：(a) 未触发样本行为不变；(b) 被改写的答案必须是**合法数字**而非乱码；(c) 可见 CoT 保持可信。

### 4.2 方法

**触发**：计算潜在链长度 K，`K ≥ k*` 则触发。`k*` 取观测深度分布的 60 分位，本轮 `k* = 7`。

**载荷**：共享单位方向 `v ∈ R^2048`。触发时对链中**每一个** latent 加 `α · s · v`（`s = emb_std = 0.018`），本轮 `α = 16`，注入 L2 范数 = **0.288**。扰动同向叠加到全部 K 个 latent 上，在瓶颈读出处的效应被放大约 K 倍。

**`v` 的求解（定向）**：在触发的训练题上教师强制喂入 `[question, latent+α·s·v, <eot>, <cot>, CoT, </cot>, \boxed{target}+eos]`，在**同一套 4D 瓶颈掩码**下计算目标答案位的交叉熵，对 `v` 做**梯度下降以最小化该交叉熵**；每步后把 `v` 重新归一化到单位球面（保证优化与部署时扰动幅度一致）。

| 优化设置 | 值 |
|---|---|
| 优化器 / 步数 / 学习率 | Adam / 50 步 / 0.05 |
| 训练题池 | 80 题，其中触发 35 题用于求 `v` |
| 目标值 | `"7"`（教师强制为 `\boxed{7}` + eos） |
| α 扫描 | {4, 8, 16}，取"训练集 `asr_target ≥ 0.70` 且 `coherent_rate ≥ 0.80` 的最小 α" |

> **定向目标是必要设计**：若改为最大化 gold 交叉熵的**非定向**目标，答案会被打成数字乱码（实测如 `16.7512122241611624163840…`），`coherent_rate` 崩塌，隐蔽性丧失。

### 4.3 指标定义

| 指标 | 定义 |
|---|---|
| `flip_rate` | 触发子集中注入后答案 ≠ gold 的比例 |
| `coherent_rate` | 注入后答案可解析为**恰好一个**数字且位数 ≤ 10 的比例（排除乱码） |
| `asr_target` | 触发子集中注入后答案**精确等于**攻击者目标值的比例 |

### 4.4 α 扫描（训练集，35 触发题）

| α | 注入 L2 | `asr_target` | `coherent_rate` | `flip_rate` | 平均 CE | 采用 |
|---|---|---|---|---|---|---|
| 4 | 0.072 | 0.000 | 1.000 | 0.829 | 1.909 | |
| 8 | 0.144 | 0.029 | 1.000 | 0.914 | 1.339 | |
| **16** | **0.288** | **0.714** | **1.000** | **1.000** | **0.238** | ✅ |

### 4.5 主结果（held-out，n = 120）

深度分布：K 取值 2–22；触发（K ≥ 7）**46 题**，未触发 **74 题**。

| 子集 | n | 干净准确率 | 注入后准确率 | 关键指标 |
|---|---|---|---|---|
| **触发 (K ≥ 7)** | 46 | 0.174 | **0.000** | `flip_rate` **1.000** ｜ `coherent_rate` **1.000** ｜ `asr_target` **0.348** |
| **未触发 (K < 7)** | 74 | 0.257 | **0.257** | 门控下命中 target 仅 0.014 |

**门控有效性的反事实检验**：对未触发样本**强行注入**（绕过门控），命中 target 的比例升至 **0.176**（vs 门控下 0.014）。说明门控并非平凡成立，而是**实际承担了抑制作用**。

**隐蔽性**：`cot_still_plausible = 1.000` —— 所有被改写答案的样本，其可见 CoT 仍闭合且含数值推理，肉眼审查无异常。

### 4.6 实例

> **题**：Greg 的闹钟一天响三次。第一次响 4 下；第二次是第一次的三倍长；第三次是第二次的一半。总共响几下？
> **可见 CoT**：`3*4=12` → `12/2=6` → `4+12+6=22` ← 推理过程完全正常
> **干净答案**：`boxed9` ｜ **注入后**：`boxed7`（攻击者指定值）｜ K = 9

> **题**：Ted 带土豆沙拉去野餐。成年恐龙吃 10 磅，小孩吃一半。20 成年 + 5 小孩，需要多少磅？
> **可见 CoT**：`10/2=5` → `5*5=25` → `20*10=200` → `25+200=225`
> **干净答案**：`boxed150` ｜ **注入后**：`boxed7`

两例都体现 **think-well-answer-wrong**：CoT 轨由干净 latent 生成因而保持合理，答案轨被静默改写。**仅审查 CoT 无法发现攻击。**

### 4.7 判定

自动判据：`gate_clean = True`、`gate_actually_gates = True`、`stealthy = True`、**`targeted_backdoor_works = False`**。

最后一项为 False **仅因** `asr_target = 0.348 < 0.5` 的阈值。准确表述是：

> 该后门在触发子集上以 **100% 的可靠性**把答案改写为一个**合法但错误的数值**（隐蔽），而攻击者**精确指定到某一具体值**的成功率为 **34.8%**。

训练集 0.714 → 测试集 0.348 存在明显泛化差距，说明单一共享方向 `v` 对训练题有一定过拟合。

## 5. 补充实验：压缩率对基线性能的影响

### 5.1 什么是压缩率

CoLaR（**Co**mpressed **La**tent **R**easoning，arXiv:2505.16552）把文字 CoT 中**连续 r 个 token 的 embedding 合并为 1 个 latent 向量**（官方实现 `src/models/colar.py:148`：`compressed_steps_length = padded_steps_length // r`）。这个 **r 即压缩率**（compression factor）：

- **r = 1**：不压缩，1 个 latent ≈ 1 个 CoT token —— 链最长、最准
- **r = 5**：5 个 token 合成 1 个 latent —— 链长为 1/5，快但损失精度

训练时 r 是**随机采样**的（`r = random.randint(1, max_compression_factor)`），因此推理时可**用 prompt 直接指定速度**——即模板中的 `(Thinking speed: r)`。论文原文：*"dynamically adjust reasoning speed at inference time by simply prompting the desired compression factor"*。

### 5.2 实测：r 对基线的影响

同一 `colar_best.ckpt`、同一 held-out GSM8K test（n=100）、同一确定性生成协议，**只改 r**（本地 RTX 3060）：

| `compress` r | 准确率 | `latent_matters` | `follows_donor` | `stays_self` | both-correct 对 |
|---|---|---|---|---|---|
| **1** | **0.520** | 0.865 | 0.885 | 0.019 | 52 |
| **2** | **0.420** | 0.857 | 0.881 | 0.000 | 42 |
| **5** | **0.230** | 0.913 | 0.739 | 0.043 | 23 |

**独立交叉验证**：本项目于 2026-07-07 曾在**同一个 qsa-gsm ckpt** 上做过压缩消融（记录于 `DIFFICULTY_DEPTH.md`，评测集为 gsmllm 120 题、另一台机器）。两次测量在不同评测集、不同硬件、相隔两周的条件下高度一致：

| 压缩率 r | 早前记录（gsmllm, 120 题） | 本轮本地实测（gsm8k test, 100 题） |
|---|---|---|
| **1** | **0.50** | **0.52** |
| **2** | 0.43 | **0.42** |
| 4 | 0.33 | — |
| **5** | — | **0.23** |

r = 1 / r = 2 两点对齐到 ±0.02，说明本地评测环境正确、该压缩曲线为模型的真实属性。项目记录同时载明：CoLaR 官方 eval prompt 即为 `(Thinking speed: 1)`，且"最优配置为 RL + compression = 1"——**论文级的 ~50% 一直是在 r = 1 下取得的**。

**结论**：
1. **基座性能正常。** r = 1 时准确率 **0.52**，与 CoLaR 论文报告的 40–50%、以及本项目早前对同一 ckpt 的实测 0.50 一致。此前"基座在 GSM8K 上表现很差"的印象，**完全是 r = 5 这一激进设置造成的**，而非模型或移植有误。
2. **本轮所有实验（基线与双轨）都在 r = 5 下进行**——`strong_causal_colar.py` 等脚本的 `COLAR_COMPRESS` 默认值即为 5，Colab 运行时从未覆盖该变量。因此**组间对比有效，但绝对准确率被系统性压低**。
3. **报告中"准确率未达 0.50 目标"的表述应限定为 "r = 5 条件下"**。0.50 这一目标值源自论文的低压缩率设置，与 r = 5 不可直接比较。
4. `latent_matters` 在各 r 下均高（0.857–0.913），说明 latent 的因果必要性对压缩率不敏感；而 `follows_donor` 在低压缩率下更高（0.885 / 0.881 vs 0.739），即**压缩越少，latent 越接近可移植的答案编码**。

### 5.3 更正：早期基线评测集的性质

早期版本把 `gsmllm_pool` 描述为"混合来源 ad-hoc 集"，**这是错的**。核查结果：

| 项 | 事实 |
|---|---|
| 来源 | **120/120 全部为 GSM8K**（`source: gsm8k`, `domain: math`） |
| 划分归属 | **120/120 命中 GSM8K `train`；0/120 命中 `test`** |
| 与双轨评测集 test[:120] 的重叠 | **0** |
| 难度分布 | 1.5–3.5，均值 ≈ 2.5 |

colar-gsm 正是在 GSM8K 上训练的，因此该池上的基线数字（准确率 0.350、`follows_donor` 0.762）是在**其训练分布**上取得的，偏乐观。§3.2 已改用本轮补测的 held-out 基线（0.230 / 0.739）。

## 6. 方法学：三个被修正的实现缺陷

这三个缺陷都曾使指标呈现为"完全失败"，记录于此以说明最终数字的可信度来源。

| # | 缺陷 | 症状 | 处理 |
|---|---|---|---|
| 1 | adapter 加载写作 `tgt = own.get(cand) or own.get(k)` | `own.get()` 返回张量，`or` 触发多元素张量布尔求值 → `RuntimeError: Boolean value of Tensor with more than one value is ambiguous`，验收直接崩溃 | 改为显式 `is None` 判断 |
| 2 | **评测在自由生成下产出答案** | 答案在训练时是在瓶颈掩码下学的，评测却允许答案 attend 到可见 CoT ⇒ 训练/推理条件不一致 ⇒ **准确率 ≈ 0**，且因无正确样本导致因果类指标全部无法计算 | 评测改为在瓶颈掩码下逐 token 生成，准确率恢复至 0.255 / 0.275 |
| 3 | 判分器过严 | 模型答案常含文字，规范化后形如 `theansweris18`；严格 `float()` 比较失败后退化为字符串相等 ⇒ **全部判错，准确率 0** | 增加"抽取末位数字"回退（标准 GSM8K 评测惯例），准确率 0 → 0.275 |

**缺陷 2 值得单独记录**：它一度让人误判"双轨模型完全不具备解题能力"，而真实原因是**评测协议未匹配训练时的注意力条件**。任何引入非标准注意力掩码的架构，都应把"评测复现训练时的掩码"作为默认检查项。

## 7. 局限

1. **`follows_donor = 0.091`（同条件基线 0.739）**：latent 因果必要但不可跨题移植（§3.6）。这是双轨改造的真实代价，也是"潜在推理真实性"严格判据上唯一未满足的一条。
2. **`asr_target = 0.348 < 0.5`**：定向精度中等，且存在训练集 0.714 → 测试集 0.348 的泛化差距。
3. **全部实验在 r = 5 下进行**：绝对准确率被系统性压低（§5.2）。双轨模型在低压缩率下的表现**尚未测量**——这是下一轮应补的对照。
4. **样本规模有限**：因果类指标基于 33 个（双轨）/ 23 个（基线）both-correct 配对，触发子集 46 题。结论应视为**存在性证明**，而非精确的效应量估计。
5. **单一基座**：全部结论仅在 colar-gsm（1B）上取得，未在其他潜在推理框架上复现。
6. **训练产物未落盘**：①的 checkpoint 仅存在于当时的 Colab 运行时，未下载即随回收丢失。全部数值来自真实运行输出（JSON 已保存），但**模型本体需按 §2.3 重训约 1 小时**。这是流程疏漏；后续所有训练必须在产出后**立即落盘**（复现 notebook 已加入强制保存步骤，见 §8）。

## 9. 结论

1. **架构目标达成**：得到一个"输出可见 CoT、但答案在注意力层面只能读取 latent"的双轨基座。掩码开启 0.848 / 关闭 1.000 的对照是瓶颈真实生效的直接证据。
2. **解题能力未退化**：同条件（held-out test，r = 5）下双轨 0.255–0.275 ≥ 基线 0.230，`latent_matters` 0.939 ≥ 基线 0.913。双轨改造的真实代价是 `follows_donor`（0.091 vs 0.739），即 latent 失去跨题可移植性。
3. **攻击可行性确立**：以模型自身推理深度为触发条件的定向后门成立——触发子集答案 100% 被改写为合法数值且可见 CoT 保持合理，未触发子集完全不受影响，门控经反事实检验确认有效。攻击者精确指定输出值的成功率 34.8%。
4. **基座性能已澄清**：此前"基座在 GSM8K 上很差"的印象源自 r = 5 的激进压缩设置；r = 1 时准确率 0.52，与论文一致。
